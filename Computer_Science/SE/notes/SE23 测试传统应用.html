传统应用测试 必考

核心问题：如何系统化地设计测试用例以最大化错误发现率——以最小的时间和精力系统性地发现不同类型的错误。

白盒测试技术 必考

  • 基本路径测试 (Basis Path Testing)：基于循环复杂度 (Cyclomatic Complexity) 计算独立路径数，确保每条路径至少执行一次
  • 循环复杂度 V(G) = E - N + 2P 或 V(G) = 判定节点数 + 1

黑盒测试技术 重点

  • 等价类划分：将输入域划分为有效等价类和无效等价类
  • 边界值分析：测试等价类的边界（最小值-1、最小值、最大值、最大值+1）
  • 判定表：对复杂逻辑条件的系统化测试
易错：循环复杂度的计算公式 $V(G) = E - N + 2P$，其中 E = 边数，N = 节点数，P = 连接组件数（通常为 1）。也可从判定节点数+1 计算。
