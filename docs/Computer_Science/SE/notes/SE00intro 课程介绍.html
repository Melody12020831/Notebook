课程基本信息

课程核心定位：帮助学生实现从「编程序」到「做软件」的重大转变。

教学目标 重点

  • 软件工程 1.0：熟悉基本概念和基本理论（过程模型、需求分析、设计、测试、项目管理）
  • 软件工程 2.0：掌握基本方法和工具（敏捷开发、CI/CD、DevOps）
  • 软件工程 3.0：了解最新工程实践（大模型驱动开发）
  • Agentic Engineering：把握软件工程研究方向

教材与参考 了解

教材：Pressman R S. 软件工程：实践者的研究方法（郑人杰译，机械工业出版社 2011）

参考书：邹欣《构建之法》（第3版）、Evans《领域驱动设计》、Martin《敏捷软件开发》

考核规则 必考

总成绩 = Project × 40% + Discussions × 20% + Final × 40% = 100%

期末考试 必考

题型：选择（多选）+ 判断 + 简答 + 画图

Discussions 重点

随机随堂测验（「学在浙大」）+ 课堂随机提问，主动回答有加分。

Bonus 了解

在 ICSE/FSE/ASE/ISSTA 等顶会发表论文，无需考试直接获得 95 分。

Project 项目 必考

项目参考推荐功能

组队规则

  • 每组 6-8 人，自由组合，组长自定
  • 组长负责：组织、分工、控制进度、分配组员得分
  • 组长奖罚：过半数组员不满 → 改选；顺利完成任务 → 总评+5；严重失误 → 总评-5

评分机制 重点

组得分 = 教师打分 × 50% +（各组长打分和）/ 组数 × 50%

教师打分 = 汇报分（汇报效果+提交物质量）+ 过程分（Git 开发过程合理性）

组员得分 = 组员贡献率 × 组得分 × 组人数

各阶段占比：需求分析 30%、项目计划 10%、系统设计 20%、原型系统 20%、系统测试 20%

3 次 Review

Review时间
需求分析与计划第十周
系统设计第十二周
原型系统与测试第十六周

提交物要求 重点

  • 完整 Git 仓库（含所有提交历史）
  • 所有文档使用 Markdown 并通过 Git 统一管理
  • 提供 AI 使用说明澄清文档及 AI 对话历史数据
易错：合理使用 AI 的原则——不是禁止 AI，也不是完全依赖 AI。底线是避免临时赶工直接用大模型生成作业提交。

教学安排 高频

理论学习四阶段：软件工程 1.0（10 周）→ 2.0（1 周）→ 3.0（4 周）→ 开源（1 周）

实验工具要求：UML (PlantUML/Mermaid)、Coding (IDE/AI coding)、Test (JUnit/JMeter)、SCM (Git/GitHub)、SQA (SonarQube)
