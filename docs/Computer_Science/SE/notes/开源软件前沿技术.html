开源运动概述

核心问题：开源不仅仅是一种开发模式——它是一场改变了整个软件产业的运动，影响从个人开发者到跨国企业。

开源核心概念 必考

  • 自由软件 vs 开源软件：自由软件强调用户自由（Richard Stallman/FSF），开源软件强调开发方法的优势
  • 开源许可证类型：宽松式 (Permissive: MIT/Apache) vs 强保护式 (Copyleft: GPL/AGPL)
  • 开源社区治理：BDFL（仁慈独裁者）模型 vs 基金会治理 vs 公司主导

开源开发模式 重点

  • 分布式协作：全球开发者异步参与——PR (Pull Request) → Code Review → Merge
  • 精英治理 (Meritocracy)：贡献决定影响力——不是头衔决定权力
  • Fork & Pull 模型：任何人都可以 Fork，贡献通过 Pull Request 回流
  • InnerSource：企业内部应用开源协作模式

开源评价维度 重点

  • 社区健康度：贡献者活跃数、Issue/PR 响应时间、新贡献者留存率
  • 项目成熟度：版本稳定性、文档质量、测试覆盖率
  • 安全评分：漏洞响应速度、安全审计历史、依赖安全性

前沿趋势

  • AI/LLM 驱动的开源开发——代码审查、Issue 分类、文档生成
  • 开源供应链安全——SBOM、依赖扫描、签名验证
  • 开放科学和数据——不仅代码开源，训练数据和研究也开放
易错：MIT 许可证 = 几乎无限制（可商用/闭源/修改）。GPL = 传染性——使用了 GPL 代码的项目也必须以 GPL 开源。Apache 2.0 = MIT + 专利授权。选错许可证有法律后果。
