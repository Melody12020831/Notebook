安全工程概述 必考

核心问题：安全不是可以事后添加的特性——必须在软件设计之初就融入系统架构。

CIA 三元组 必考

  • 机密性 (Confidentiality)：信息只被授权者访问
  • 完整性 (Integrity)：信息不被未经授权修改
  • 可用性 (Availability)：授权用户在需要时可访问

安全威胁类型

  • 拦截 (Interception)：未授权方访问数据
  • 中断 (Interruption)：服务变得不可用
  • 篡改 (Modification)：未授权方拦截并修改数据
  • 伪造 (Fabrication)：未授权方插入假冒数据

安全设计原则 重点

  • 最小权限 (Least Privilege)：每个组件只授予完成工作所需的最小权限
  • 纵深防御 (Defense in Depth)：多层安全控制
  • 默认安全 (Secure by Default)：默认配置即为安全配置
  • 安全失败 (Fail Secure)：失败时进入安全状态而非暴露漏洞
易错：安全三要素 CIA —— Confidentiality (机密) / Integrity (完整) / Availability (可用)。考试常考各要素对应哪种攻击。
