界面设计基础 必考

核心问题：接口是软件的「脸面」——无论内部功能多强大，糟糕的 UI 会让用户拒绝使用。

黄金规则 (Golden Rules) 必考

  1. 将控制权交给用户 (Place the User in Control)
  2. 减少用户记忆负担 (Reduce the User's Memory Load)
  3. 保持界面一致 (Make the Interface Consistent)

具体设计原则 重点

  • 以不显眼的模式定义交互——避免强制模态对话框
  • 提供灵活交互——支持键盘/鼠标/触摸多种方式
  • 允许中断和撤销——给用户反悔的机会
  • 使用有意义的隐喻——利用用户已有知识
  • 保持界面简洁——减少视觉噪音
  • 提供有意义的反馈——每次操作都有确认

UI 设计过程 重点

  1. 用户分析（谁在使用？经验水平？）
  2. 任务分析（用户要完成什么？）
  3. 界面原型创建
  4. 用户评估和测试

Web 界面设计

响应式设计；导航清晰；内容优先（content is king）；加载速度；可访问性 (a11y)。

易错：用户界面设计不是纯粹的「美术」问题——它是交互设计+认知心理学+软件工程的交叉学科。美观不等于好用。
