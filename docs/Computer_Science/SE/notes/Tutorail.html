Tutorial 概述

核心目标：本 Tutorial 是对课程关键概念的综合实践回顾——帮助学生在期末前巩固核心知识点。

关键回顾领域

  • 软件过程模型：瀑布/增量/螺旋/UP/敏捷——各模型的选择标准
  • 需求工程：七步法、用例建模、分析模型四视图
  • 设计概念：SOLID 原则、架构风格、构件设计、设计模式
  • 质量保证：评审技术、SQA、测试策略与层次
  • 项目管理：估算 (COCOMO)、调度 (CPM/甘特图)、风险分析

考试准备要点 必考

  • 选择/判断——概念辨析（如：验证 vs 确认、黑盒 vs 白盒、内聚 vs 耦合）
  • 简答——定义+举例（如：什么是基线？举例说明变更控制流程）
  • 画图——UML 图（用例图/类图/状态图/序列图）和架构图
  • 综合——跨章节串联概念（如：从需求→设计→测试的完整流程）
核心提醒：软件工程不是背诵定义——而是理解为什么需要这个过程。每学一个概念问自己：如果没有它，软件项目会出什么问题？
