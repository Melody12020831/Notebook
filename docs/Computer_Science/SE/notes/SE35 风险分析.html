风险管理概述 必考

核心问题：软件开发充满不确定性——风险管理不是消除不确定性，而是识别、评估和减轻不确定性的影响。

风险分类 重点

  • 项目风险：影响项目进度和资源（人员流失、预算削减）
  • 技术风险：影响产品质量和交付（技术选型失败、性能不达标）
  • 商业风险：影响产品价值（市场变化、竞争对手产品）

风险管理步骤 必考

  1. 识别 (Identification)：列出所有可能的风险——使用风险检查表、头脑风暴
  2. 分析 (Analysis)：评估每个风险的概率和影响——建立风险矩阵
  3. 计划 (Planning)：为高影响/高概率风险制定缓解和应急计划
  4. 监控 (Monitoring)：持续跟踪和重新评估风险

RMMM 策略 重点

缓解 (Mitigation)：降低风险发生概率。 监控 (Monitoring)：跟踪风险指标。 管理 (Management)：风险发生时的应急计划。

易错：风险是「可能发生的问题」，不是「已经发生的问题」。已经发生的是问题/危机，不是风险。风险管理关注未来。
