设计模式基础 必考

核心问题：如何复用已被验证的设计方案而非每次从零开始——这就是设计模式的价值。

设计模式：对在特定上下文中重复出现的问题的通用可复用解决方案。

模式描述模板

模式名 → 意图 → 别名 → 动机 → 适用性 → 结构 → 参与者 → 协作 → 后果 → 实现 → 示例代码 → 已知使用 → 相关模式。

常用模式举例 重点

  • Singleton（单例）：确保一个类只有一个实例，提供全局访问点
  • Observer（观察者）：一对多依赖，当对象状态改变时通知所有依赖者
  • Strategy（策略）：定义一系列算法，将每个封装起来并使它们可以互换
  • Factory Method（工厂方法）：定义创建对象的接口，由子类决定实例化哪个类
易错：模式不是可复制的代码片段——而是经过验证的设计思想和解决方案结构。盲目套用模式可能比不用模式更糟糕。
